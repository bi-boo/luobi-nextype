# 落笔 Nextype — Mac 端产品需求文档

## 产品定位与目标

落笔 Nextype 是一款跨设备剪贴板同步应用，核心场景是「手机打字，电脑上屏」——用户在手机端通过语音输入法等方式输入文字，内容实时同步到 Mac 端并自动粘贴到当前光标位置。本质上是将手机变成电脑的无线键盘。

Mac 端（Tauri 2.x 版本）是从旧版 Electron 桌面端迁移而来，保持了功能对齐的同时大幅降低了资源占用。应用以菜单栏常驻方式运行，无主窗口，通过托盘图标提供所有入口。

## 核心功能清单

### 1. 设备配对

- **配对码生成**：生成 4 位数字配对码，优先使用易记忆模式（AABB、ABAB、连续递增/递减、回文 ABBA、AAAB/ABBB、AABA/ABAA 共 7 种模式），冲突时自动切换为纯随机模式
- **配对码有效期**：60 秒自动过期，过期后需重新生成
- **二维码配对**：将配对信息（配对码、设备 ID、中继服务器地址）编码为 JSON 并生成二维码 Data URL（PNG 格式），供手机端扫码
- **配对码冲突处理**：中继服务器返回 `code_conflict` 时，自动以 `forceRandom=true` 重新生成
- **配对码自动注册**：生成配对码后自动向中继服务器注册，重连后自动恢复注册
- **配对完成通知**：配对成功后发送系统通知（macOS Notification），并自动将对方加入信任设备列表

### 2. 剪贴板同步

- **写入剪贴板**：通过 `tauri-plugin-clipboard-manager` 将接收到的文本写入系统剪贴板
- **自动粘贴（Cmd+V）**：通过 AppleScript 模拟 `keystroke "v" using command down`，写入剪贴板后延迟 100ms 执行
- **粘贴+回车**：先执行粘贴，延迟 300ms 后通过 AppleScript 模拟 `keystroke return`
- **仅复制**：只写入剪贴板，不执行粘贴操作
- **后缀追加**：粘贴动作可追加 `btn1_suffix`，粘贴+回车动作可追加 `btn2_suffix`
- **粘贴后自动清空**：可配置在粘贴完成 200ms 后自动清空剪贴板（隐私保护）
- **加密传输**：支持 CryptoJS 兼容的 AES-256-CBC 解密（OpenSSL 格式：Salted__ + salt + 密文，EVP_BytesToKey 密钥派生），使用发送方 deviceId 作为密码
- **ACK 回执**：剪贴板内容处理成功后，向发送方回传 `ack` 消息

### 3. 快捷键远程控制

- **发送（send）**：触发手机端「发送/回车」按钮
- **插入（insert）**：触发手机端「插入」按钮
- **清空（clear）**：触发手机端「清空」按钮
- **模拟点击（tap）**：向手机端发送指定坐标的点击指令
- **模拟长按（longpress）**：
  - 按下时发送 `touch_down` + 坐标
  - 按住期间每 500ms 发送 `touch_heartbeat` + 坐标
  - 释放时发送 `touch_up` + 坐标
  - 可配置释放后自动发送 `insert` 指令（延时可调，默认 300ms）
- **防抖机制**：所有快捷键触发间隔 500ms 防抖

### 4. 快捷键录入

- **原生 Fn 键支持**：通过 macOS NSEvent `addLocalMonitorForEvents` 捕获 keyDown + flagsChanged 事件，支持录入含 Fn 键的快捷键组合
- **支持的快捷键类型**：
  - 纯修饰键组合（如 Fn、Fn+Ctrl）
  - Fn + 主键（如 Fn+1）
  - Fn + 修饰键 + 主键（如 Fn+Ctrl+1）
  - 常规修饰键 + 主键（不含 Fn，走 tauri-plugin-global-shortcut）
- **实时修饰键显示**：录入过程中实时显示当前按下的修饰键（macOS 符号风格：⌃⌥⇧⌘）
- **Escape 取消**：录入过程中按 Escape 清除当前快捷键
- **非 macOS 平台 fallback**：保留 JS keydown/keyup 录入逻辑

### 5. 屏幕坐标配置

- **动态多维度配置**：支持为手机的不同屏幕尺寸（折叠态、展开态、横屏等）分别配置点击/长按坐标
- **自动捕获新尺寸**：手机端上报 `device_info` 或 `screen_changed` 时，自动识别新屏幕尺寸并添加默认配置（X 取屏幕宽度中点，Y 取屏幕高度 90%）
- **坐标匹配策略**（三级容错）：
  1. 精确匹配：宽高差值 < 1px
  2. 比例匹配：宽高比差值 < 0.1，取最接近的配置
  3. 兜底匹配：使用第一个配置
- **点击坐标和长按坐标独立配置**：各自有独立的坐标列表和持久化存储

### 6. 托盘菜单

- **菜单项**：
  - 连接手机（打开偏好设置 → 连接手机标签页）
  - 偏好设置（打开偏好设置 → 基础设置标签页）
  - 在线设备（标题 + 设备列表，显示设备名称，无设备时显示「暂无设备」）
  - 日志（打开日志窗口）
  - 退出 落笔 Nextype
- **左键点击**：直接弹出菜单（`show_menu_on_left_click: true`）
- **图标状态**：
  - 未连接状态：`iconTemplate.png`
  - 已连接状态：`iconConnectedTemplate.png`（有设备在线时自动切换）
  - 使用 macOS Template 图标（自动适配深色/浅色菜单栏）
- **动态更新**：设备上线/下线/超时时自动重建菜单和切换图标

### 7. 偏好设置窗口

5 个标签页，通过侧边栏切换：

#### 7.1 基础设置
- 开机启动（通过 `tauri-plugin-autostart` 实现 LaunchAgent）
- 菜单栏图标显示/隐藏（至少保留一个图标的互斥保护）
- Dock 栏图标显示/隐藏（通过 `ActivationPolicy::Regular/Accessory` 切换）
- 清空剪贴板（粘贴后自动清空）
- 辅助功能权限检查（点击跳转系统设置，窗口获得焦点时自动重新检查）

#### 7.2 连接手机
- 配对码显示与刷新（60 秒倒计时）
- 已连接设备列表（区分 Android/iOS 图标，支持解除配对）
- 下载手机端应用按钮（Android 版 / iOS 版）

#### 7.3 使用数据
- 同步次数（累计）
- 今日同步字数
- 累计同步字数
- 启用/禁用统计功能开关

#### 7.4 远程控制
- 快捷键配置（发送、插入、清空）
- 模拟点击配置（快捷键 + 多屏幕尺寸坐标）
- 模拟长按配置（快捷键 + 多屏幕尺寸坐标 + 自动插入开关和延时）

#### 7.5 关于和反馈
- 应用 Logo、名称、版本号（动态获取）
- 检查更新（跳转官网）
- 我要反馈（跳转邮件）
- 版权信息

### 8. 引导流程（Onboarding）

4 步引导窗口，首次启动自动显示（通过 `hasRunBefore` 标记持久化）：

1. **欢迎页**：介绍产品核心价值「手机打字，电脑上屏」
2. **权限页**：检查辅助功能权限，自动触发权限请求弹窗，未授权时自动打开系统设置
3. **隐私安全页**：说明端到端加密、不上传内容、无需注册
4. **立即体验页**：生成二维码供扫码配对

支持左右箭头导航和底部圆点指示器。

### 9. 日志系统

- **四层架构**：
  1. 控制台输出（`tracing_subscriber::fmt`）
  2. 文件写入（`~/Library/Logs/nextype-tauri/clipboard-sync.log`，超过 10MB 自动轮转）
  3. 内存缓存（最近 1000 条，供前端实时查看）
  4. 前端推送（通过 Tauri Event `new_log_entry` 实时推送）
- **日志窗口**：
  - Grid 布局（时间 | 日志类型 | 消息类型 | 消息内容）
  - 支持刷新、导出（保存文件或复制到剪贴板）、清空
  - 实时滚动到底部
  - 日志级别颜色区分（INFO 蓝色、WARN 橙色、ERROR 红色、DEBUG 紫色）

### 10. 使用统计

- **统计指标**：累计同步次数、累计同步字符数、今日同步字符数
- **日期重置**：每日首次访问时自动重置今日统计
- **持久化**：通过 `tauri-plugin-store` 存储在 `stats.json`
- **实时更新**：每次粘贴操作后通过 `stats_updated` 事件通知前端
- **可禁用**：关闭后不再记录统计数据，返回空数据

### 11. 系统集成

- **开机启动**：通过 `tauri-plugin-autostart` 注册 macOS LaunchAgent，启动参数 `--minimized`
- **Dock 图标**：通过 `ActivationPolicy` 切换 Regular（显示）/ Accessory（隐藏），切换时保持已打开窗口可见
- **菜单栏图标**：通过 `tray.set_visible()` 控制显示/隐藏
- **单实例保护**：通过 `tauri-plugin-single-instance` 防止重复启动，重复启动时聚焦现有窗口
- **窗口关闭拦截**：点击窗口关闭按钮不退出应用，仅隐藏窗口（通过 `ExitRequested` + `prevent_exit`）
- **Accessory 模式窗口前置**：使用 `set_always_on_top(true)` → `set_focus()` → `set_always_on_top(false)` 三步法
- **多显示器支持**：新建窗口时通过 CGEvent 获取鼠标物理坐标，定位到鼠标所在显示器居中

### 12. Electron 数据迁移

- **配置迁移**：从 `~/Library/Application Support/落笔 Nextype/clipboard-sync-config.json` 读取 Electron 配置，逐字段迁移
- **中继服务器地址升级**：迁移时自动将旧的 `ws://<your-server-ip>` 升级为配置的中继服务器地址
- **统计数据迁移**：一次性叠加 Electron 版本的历史数据（硬编码：4140 次同步、253564 字符），通过 `electron_data_migrated` 标记防止重复迁移
- **信任设备迁移**：从 Electron 配置中读取 `trustedDevices` 数组并合并

### 13. 中继连接与网络管理

- **WebSocket 中继**：通过 WSS 连接中继服务器（默认 `wss://nextypeapi.yuanfengai.cn:8443`）
- **自动重连**：断线后 5 秒间隔重试，最多 10 次
- **网络变化监测**：每 5 秒检测本机 IP 变化，网络切换或恢复时自动重置重连计数并重新连接
- **心跳机制**：每 10 秒发送心跳（包含系统闲置时间，通过 `ioreg` 读取 HIDIdleTime）
- **客户端超时检查**：每 60 秒扫描一次，120 秒无活动的客户端标记为离线
- **信任列表同步**：注册成功后自动请求同步信任列表（`SyncTrustList`），合并到本地

### 14. 深色/浅色模式

- 前端通过 CSS 变量（`style.css`）支持 `prefers-color-scheme` 自动切换
- 托盘图标使用 macOS Template 图标自动适配

## 用户流程

### 首次使用流程

1. 用户首次启动应用 → 自动弹出引导窗口
2. 引导第 1 页：了解产品功能
3. 引导第 2 页：授予辅助功能权限（自动弹出系统权限弹窗）
4. 引导第 3 页：了解隐私安全保障
5. 引导第 4 页：扫描二维码完成配对
6. 关闭引导窗口 → 应用常驻菜单栏运行

### 日常使用流程

1. 应用在菜单栏常驻运行，自动连接中继服务器
2. 手机端打开落笔 App，自动连接（已配对设备）
3. 用户在手机端输入文字 → 点击「同步」/「发送」
4. Mac 端收到加密内容 → 解密 → 写入剪贴板 → 自动粘贴到当前光标位置
5. 统计数据自动更新

### 远程控制流程

1. 用户在偏好设置 → 远程控制中配置快捷键和坐标
2. 手机端连接后上报屏幕尺寸，Mac 端自动捕获并添加坐标配置
3. 用户按下快捷键 → Mac 端匹配坐标 → 向手机端发送指令
4. 长按快捷键：按下发送 touch_down → 持续发送心跳 → 释放发送 touch_up → 可选自动插入

## 各功能详细交互说明

### 配对码交互

- 进入「连接手机」标签页时自动生成配对码
- 配对码以大号等宽字体居中显示，下方显示倒计时
- 倒计时归零后显示「配对码已过期，请刷新」
- 点击「刷新配对码」按钮重新生成
- 配对成功后显示「DONE」和设备名称，2 秒后自动重新生成新配对码

### 图标互斥保护

- 菜单栏图标和 Dock 图标不能同时关闭
- 当一个已关闭时，尝试关闭另一个会被阻止并显示提示「至少需要保留一个图标用于访问应用」
- 提示 2 秒后自动消失

### 辅助功能权限

- 基础设置页显示权限状态开关（只读）
- 点击整行跳转系统辅助功能设置
- 窗口获得焦点时自动重新检查权限状态
- 引导流程中自动触发权限请求弹窗

## 废弃与未使用代码

### Rust 后端

- **[残留代码]** `src-tauri/src/utils/config.rs:33` — `port: u16` 字段
  - 原因：这是 Electron 版本的局域网直连端口配置，Tauri 版本已完全使用中继服务器通信，`port` 字段在后端代码中除了 Electron 迁移逻辑外无任何读取/使用
  - 建议：保留用于 Electron 迁移兼容，但可标注为 deprecated

- **[残留代码]** `src-tauri/src/utils/config.rs:51-53` — `min_poll_interval` 和 `max_poll_interval` 字段
  - 原因：这是 Electron 版本的轮询间隔配置，Tauri 版本使用 WebSocket 实时推送，这两个字段在整个后端代码中无任何读取/使用
  - 建议：可安全移除（注意保持 JSON 反序列化兼容）

- **[残留代码]** `src-tauri/src/utils/config.rs:63` — `auto_update: bool` 字段
  - 原因：Tauri 版本未实现自动更新功能，该字段仅在 Electron 迁移时被写入，后续无任何读取
  - 建议：保留用于迁移兼容

- **[残留注释]** `src-tauri/src/lib.rs:184-185` — 关于事件中转的注释
  - 原因：说明剪贴板同步消息已由 relay_client.rs 直接处理，不再需要通过 app.listen 中转。这是架构变更的历史记录
  - 建议：可保留作为架构说明

- **[残留注释]** `src-tauri/src/commands/system.rs:7-8` — 关于权限函数不再重复的注释
  - 原因：说明 `has_accessibility_permission` 和 `open_accessibility_settings` 已在 clipboard.rs 中实现
  - 建议：可保留作为代码导航提示

### 前端

- **[残留代码]** `src/preferences.html:458-487` — `parseDeviceInfo` 函数中的 userAgent/model 解析逻辑
  - 原因：函数中大量解析 `userAgent` 和 `model` 字段的逻辑（iOS 版本号、Android 型号等），但当前 Tauri 版本的设备信息直接使用 `device.name` 字段，这些解析分支实际上很少被触发
  - 建议：可简化，仅保留 `device.name` 的处理

### 统计数据迁移

- **[残留代码]** `src-tauri/src/commands/stats.rs:141-142` — 硬编码的 Electron 历史数据
  - 原因：`ELECTRON_TOTAL_PASTES = 4140` 和 `ELECTRON_TOTAL_CHARS = 253564` 是开发者个人的 Electron 版本使用数据，硬编码在代码中用于一次性迁移
  - 建议：迁移完成后（`electron_data_migrated = true`），这段代码不会再执行，但硬编码个人数据不够通用。如果是面向其他用户发布，应改为从 Electron 配置文件中读取实际统计数据
